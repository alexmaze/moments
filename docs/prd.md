# 私有部署社交平台需求文档（MVP v1）

## 1. 文档信息

- 文档名称：私有部署社交平台需求文档（MVP v1）
- 文档类型：产品需求文档（PRD）
- 文档目标：明确第一阶段 MVP 的产品范围、核心能力、约束条件与系统设计边界，为后续架构设计、数据建模、接口设计和开发排期提供依据
- 产品形态：开源、支持私有部署的小圈子社交平台
- 核心参考体验：微信朋友圈的“发动态 + 浏览动态 + 点赞评论”主链路，但不追求完整复刻

---

## 2. 产品背景

当前目标是构建一个适用于私有部署场景的轻量社交平台，主要服务于固定小圈子用户。平台强调：

- 部署简单，可由个人或小团队自行部署
- 功能克制，优先保证核心链路可用
- 不引入复杂权限体系与社交关系体系
- 后续可扩展 AI 自动总结、内容分析、互动分析等能力

与面向公网的大型社交平台不同，本产品默认部署在相对可信的小范围环境中，因此产品策略上采取：

- 少规则
- 少限制
- 少复杂权限
- 强调可用性和可扩展性

---

## 3. 产品目标

### 3.1 第一阶段目标

交付一个可实际使用的 MVP，满足以下核心需求：

1. 支持多个账号登录使用
2. 支持发布文字、图片、视频混合内容的帖子
3. 支持所有用户查看全部帖子
4. 支持对帖子进行点赞与评论
5. 支持私有部署
6. 为后续 AI 分析类能力预留扩展空间

### 3.2 非目标

以下能力不属于当前 MVP 范围：

- 好友关系
- 关注关系
- 权限隔离
- 分组可见
- 评论回复
- 消息通知
- 内容审核
- 搜索
- 标签 / 话题
- 转发 / 分享
- 实时聊天
- 推荐流
- AI 自动总结 / 自动分析功能本身

---

## 4. 产品定义

本产品可以定义为：

> 一个开源、支持私有部署的小圈子动态社交平台。用户可使用多个账号登录，发布包含文字、图片、视频的混合帖子，所有成员默认互相可见，并可对帖子进行点赞和评论。

## 5. 用户角色

当前 MVP 仅定义一种前台用户角色：

### 5.1 普通用户

能力包括：

- 登录系统
- 浏览全部帖子
- 发布帖子
- 删除自己的帖子
- 点赞 / 取消点赞
- 评论帖子
- 删除自己的评论
- 查看其他用户发布的帖子
- 查看帖子详情

### 5.2 管理能力

当前版本不建设复杂后台权限体系。若部署方需要账号创建与维护，可在实现阶段通过以下任一种方式满足：

- 初始化脚本创建账号
- 命令行创建账号
- 预留最简后台管理入口

该部分不作为当前产品能力重点。

## 6. 产品原则

### 6.1 极简范围原则

只实现最核心的社交动态主链路：

- 发帖
- 看帖
- 点赞
- 评论

### 6.2 全员可见原则

系统不做好友、关注、圈层隔离，所有用户默认可见全部帖子。

### 6.3 媒体自由组合原则

帖子是内容容器，媒体是有序附件。帖子支持：

- 纯文字
- 文字 + 图片
- 文字 + 视频
- 文字 + 图片 + 视频混发
- 纯图片
- 纯视频
- 图片 + 视频混发

系统不通过“帖子类型”限制媒体组合。

### 6.4 平台少限制原则

作为私有部署产品，系统在产品层面不主动设置严格业务限制：

- 不限制单帖媒体数量
- 不限制单个视频文件大小

资源边界由部署环境和基础设施能力决定。

### 6.5 展示裁剪而非数据裁剪原则

系统不限制用户可发布的内容规模，但在展示层可采取分页、懒加载、预览裁剪等性能优化手段。

### 6.6 为 AI 扩展预留原则

数据模型、媒体结构、行为记录和异步处理链路应为后续 AI 能力预留空间，例如：

- 内容摘要
- 图片理解
- 视频总结
- 用户行为分析
- 热点总结

## 8. 用户场景

### 8.1 发布动态

用户登录后，可以输入一段文字，并上传任意数量的图片和视频，以混合形式发布一条帖子。

### 8.2 浏览动态流

用户进入首页后，可以按时间倒序浏览所有用户发布的帖子。

### 8.3 互动

用户浏览帖子时，可以对帖子点赞，也可以发表评论。

### 8.4 查看详情

当帖子媒体较多或评论较多时，用户可进入帖子详情页查看完整内容与完整评论。

### 8.5 查看个人发帖

用户可进入某个用户主页，查看该用户已发布的帖子列表。

## 9. 页面与信息架构

### 9.1 登录页

#### 功能
- 用户名输入
- 密码输入
- 登录按钮

#### 说明
当前 MVP 不要求复杂注册流程。账号可由部署方预创建，或在实现阶段按需要补充最简注册功能。

### 9.2 首页 Feed

#### 功能
- 按时间倒序展示全部帖子
- 显示发帖人信息
- 显示发布时间
- 显示文本内容
- 显示媒体预览
- 显示点赞数
- 显示评论数
- 点赞 / 取消点赞
- 展示部分评论预览
- 进入帖子详情

#### 说明
首页适合作为平台默认入口。

### 9.3 发帖页 / 发帖组件

#### 功能
- 输入文字内容
- 上传图片
- 上传视频
- 支持图片视频混发
- 支持媒体排序
- 发布帖子

#### 说明
发布逻辑建议采用“两阶段模型”：先上传媒体，再提交帖子内容与媒体引用关系。

### 9.4 帖子详情页

#### 功能
- 完整展示帖子文本
- 完整展示全部媒体
- 完整展示全部评论

#### 说明
首页以预览为主，详情页负责承载完整内容。

### 9.5 用户主页

#### 功能

- 显示该用户发布的帖子列表

## 10. 功能需求

## 10.1 账号与用户

### 10.1.1 登录

系统应支持多个账号登录。

#### 输入
- 用户名
- 密码

#### 输出
- 登录成功后进入首页
- 返回会话凭证或访问令牌

### 10.1.2 用户资料

用户资料包含至少以下字段：

- 用户 ID
- 用户名
- 昵称
- 头像
- 创建时间

### 10.1.3 可见性规则

当前版本中：

- 所有用户默认可以查看全部帖子
- 不做额外权限控制
- 不做内容可见范围配置

---

## 10.2 帖子

### 10.2.1 发帖

用户可发布一条帖子。帖子由以下内容组成：

- 文本内容（可为空）
- 媒体列表（可为空，但文本和媒体至少有一项存在）

#### 支持的组合
- 纯文字
- 纯图片
- 纯视频
- 图文
- 视频 + 文字
- 图片 + 视频
- 图片 + 视频 + 文字

#### 约束
- 帖子至少包含文本或至少一个媒体
- 媒体顺序需被保留

### 10.2.2 删除帖子

用户可删除自己发布的帖子。

#### 规则
- 仅作者本人可删除
- 删除后帖子不再对前台可见
- 推荐采用软删除实现，保留数据以便审计、恢复或后续分析

### 10.2.3 浏览帖子列表

系统应提供帖子列表能力：

- 首页全量动态流
- 某个用户的帖子列表

#### 排序规则
- 按创建时间倒序
- 当前版本不做推荐排序

### 10.2.4 查看帖子详情

帖子详情页需完整展示：

- 作者信息
- 发布时间
- 文本内容
- 全部媒体
- 点赞信息
- 全部评论

---

## 10.3 媒体资源

### 10.3.1 图片

系统应支持图片上传与展示。

#### 要求
- 支持多个图片
- 支持与视频混发
- 支持按顺序展示

### 10.3.2 视频

系统应支持视频上传与展示。

#### 要求
- 支持多个视频
- 支持与图片混发
- 支持按顺序展示
- 产品层不限制视频大小

### 10.3.3 媒体数量与大小策略

产品层面不限制：

- 单帖媒体数量
- 单个视频大小

实际边界由部署环境决定，例如：

- 反向代理配置
- 网关配置
- 对象存储能力
- 磁盘容量
- 上传超时配置
- 浏览器能力

### 10.3.4 媒体顺序

同一帖子中的媒体应保留上传顺序。展示时应按该顺序渲染，不自动按图片或视频分类重排。

### 10.3.5 媒体展示策略

#### 首页 Feed
- 展示媒体预览
- 当媒体数量过多时可只展示部分内容并提供“查看全部”入口
- 可采用懒加载机制

#### 帖子详情页
- 支持完整展示全部媒体
- 支持按顺序浏览

### 10.3.6 上传建议

虽然不是产品显式功能，但实现上建议采用：

- 媒体先上传
- 帖子后提交

即两阶段流程：

1. 上传图片 / 视频，得到媒体资源标识
2. 提交帖子文本与媒体 ID 列表

原因：

- 更适合大文件上传
- 支持失败重试
- 便于断点续传
- 有利于异步处理与后续 AI 分析

---

## 10.4 点赞

### 10.4.1 点赞帖子

用户可以对任意帖子点赞。

#### 规则
- 一个用户对同一帖子只能点一次赞
- 重复点赞请求应具备幂等性

### 10.4.2 取消点赞

用户可以取消自己对帖子点的赞。

### 10.4.3 点赞展示

当前版本至少需要展示：

- 点赞总数

可选扩展：

- 点赞用户名单

MVP 中优先保证点赞数可见。

---

## 10.5 评论

### 10.5.1 发表评论

用户可以对帖子发表评论。

#### 规则
- 评论内容为纯文本
- 当前版本仅支持一级评论
- 不支持评论回复

### 10.5.2 删除评论

用户可以删除自己的评论。

#### 规则
- 仅评论作者本人可删除
- 推荐采用软删除实现

### 10.5.3 评论展示

#### 首页 Feed
- 可展示最新评论预览
- 建议只展示少量评论，避免卡片过长

#### 帖子详情页
- 展示完整评论列表

### 10.5.4 评论排序

建议默认按创建时间正序展示，便于阅读。

---

## 11. 交互规则

### 11.1 帖子内容校验

发帖时应保证：

- 纯空文本且无媒体时禁止发布
- 文本允许为空
- 仅上传媒体时允许发布

### 11.2 帖子展示顺序

- 所有帖子按时间倒序排列
- 新帖子优先显示

### 11.3 媒体渲染

- 图片与视频混合展示
- 严格按顺序输出
- 列表页可做预览裁剪
- 详情页可完整查看

### 11.4 点赞幂等

- 已点赞再次点赞，不应产生重复记录
- 未点赞取消点赞，不应报严重错误

### 11.5 删除规则

- 用户仅能删除自己的帖子
- 用户仅能删除自己的评论

## 12. 数据模型建议

以下为建议性数据模型，供后续系统设计使用。

### 12.1 User

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigint / uuid | 用户 ID |
| username | string | 用户名，唯一 |
| password_hash | string | 密码哈希 |
| nickname | string | 昵称 |
| avatar_url | string | 头像地址 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |
| deleted_at | datetime nullable | 软删除时间 |

### 12.2 Post

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigint / uuid | 帖子 ID |
| user_id | fk | 作者 ID |
| content_text | text | 文本内容 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |
| deleted_at | datetime nullable | 软删除时间 |

说明：
- 不建议使用 `post_type` 字段限制帖子形式
- 帖子是否为纯文本、图文、视频混发，应由文本与媒体关联共同决定

### 12.3 MediaAsset

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigint / uuid | 媒体资源 ID |
| uploader_user_id | fk | 上传者 ID |
| media_type | enum | image / video |
| storage_key | string | 存储键 |
| media_url | string | 访问地址 |
| cover_url | string nullable | 视频封面 |
| mime_type | string | MIME 类型 |
| file_size | bigint | 文件大小 |
| width | int nullable | 宽度 |
| height | int nullable | 高度 |
| duration | int nullable | 时长（秒或毫秒） |
| upload_status | enum | uploading / uploaded / failed |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

说明：
- 媒体先独立存在，不必须在上传时直接绑定帖子
- 便于支持先上传、后发帖

### 12.4 PostMediaRelation

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigint / uuid | 关联 ID |
| post_id | fk | 帖子 ID |
| media_asset_id | fk | 媒体资源 ID |
| sort_order | int | 排序序号 |
| created_at | datetime | 创建时间 |

说明：
- 用于维护帖子与媒体的关联
- 通过 `sort_order` 保留媒体顺序

### 12.5 PostLike

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigint / uuid | 点赞 ID |
| post_id | fk | 帖子 ID |
| user_id | fk | 用户 ID |
| created_at | datetime | 创建时间 |

约束建议：
- unique(post_id, user_id)

### 12.6 PostComment

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigint / uuid | 评论 ID |
| post_id | fk | 帖子 ID |
| user_id | fk | 评论用户 ID |
| content | text | 评论内容 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |
| deleted_at | datetime nullable | 软删除时间 |

说明：
- 当前版本不设计 `parent_comment_id`
- 即只支持一级评论

### 12.7 EventLog（建议预留）

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigint / uuid | 事件 ID |
| user_id | fk | 操作用户 |
| event_type | string | 如 post_created / post_liked / comment_created |
| target_type | string | post / comment / media |
| target_id | string | 目标 ID |
| payload | json | 扩展信息 |
| created_at | datetime | 创建时间 |

说明：
- 不是 MVP 必须项，但强烈建议预留
- 便于后续做 AI 总结、行为分析、活跃度分析

## 14. 性能与工程策略

## 14.1 列表与详情分层

### 首页 Feed
- 以浏览效率为主
- 展示帖子摘要和媒体预览
- 评论展示预览
- 对媒体较多的帖子可做折叠与懒加载

### 帖子详情页
- 以完整内容承载为主
- 完整展示全部媒体和评论

## 14.2 上传链路

由于产品层不限制媒体数量和视频大小，建议实现上支持：

- 分片上传
- 断点续传
- 客户端直传对象存储（可选）
- 服务端异步确认

## 14.3 大资源处理原则

系统不从产品规则上限制资源规模，但应在工程层采取措施保证可用性，例如：

- 分页
- 懒加载
- 缩略图
- 视频封面
- 异步处理
- CDN / 对象存储接入

---

## 15. 非功能需求

## 15.1 部署方式

产品需支持私有部署，建议至少支持：

- Docker Compose
- Kubernetes（后续可补充 Helm Chart）

## 15.2 存储支持

媒体资源建议支持以下后端之一：

- 本地文件系统
- MinIO
- S3 兼容对象存储
- 其他对象存储（如 OSS / COS）

## 15.3 可配置性

以下能力应配置化：

- 数据库连接
- 对象存储配置
- 上传路径
- 访问域名
- 反向代理大小限制
- 站点基础信息

## 15.4 安全性

当前版本最低要求：

- 密码安全存储（哈希）
- 基础身份校验
- 接口鉴权
- 基础输入校验
- 防止未授权删除他人内容

## 15.5 可观测性

建议具备：

- 基础应用日志
- 错误日志
- 上传失败日志
- 请求追踪能力（可选）

## 15.6 备份恢复

建议部署方案支持：

- 数据库备份
- 媒体存储备份
- 配置备份

## 16. AI 扩展预留

虽然 AI 功能不属于当前版本，但设计上建议预留以下能力：

### 16.1 结构化内容输入

帖子应拆分为：

- 文本内容
- 图片集合
- 视频集合
- 媒体顺序关系

便于后续做多模态分析。

### 16.2 事件日志

记录用户关键行为：

- 发帖
- 点赞
- 评论
- 删除帖子
- 删除评论

便于后续做：

- 活跃度分析
- 小圈子周报
- 热门内容总结
- 用户互动分析

### 16.3 异步任务机制

建议系统具备异步任务能力，为未来接入以下能力做好准备：

- 视频封面生成
- 媒体元数据提取
- 图片理解
- 视频理解
- 自动摘要
- 群体内容聚合分析

## 17. 验收标准

当满足以下条件时，可视为 MVP 达成：

### 17.1 用户能力
- 可创建并登录多个账号
- 登录后可正常使用系统

### 17.2 帖子能力
- 可发布纯文字帖子
- 可发布纯图片帖子
- 可发布纯视频帖子
- 可发布图文混合帖子
- 可发布图片与视频混发帖子
- 可按顺序展示媒体
- 可删除自己的帖子

### 17.3 浏览能力
- 首页可按时间倒序展示全部帖子
- 可查看帖子详情
- 可查看某个用户的帖子列表

### 17.4 互动能力
- 可对帖子点赞和取消点赞
- 可对帖子发表评论
- 可删除自己的评论
- 可查看评论列表

### 17.5 私有部署能力
- 能在私有环境中完成部署
- 媒体可正常上传与访问

## 19. 一句话总结

该 MVP 的目标不是做一个完整的微信或微博，而是做一个：

> 开源、支持私有部署、面向小圈子使用的轻量动态社交平台，支持多账号、文字图片视频混发、全员可见、点赞评论，并为后续 AI 自动总结分析能力预留结构化基础。
